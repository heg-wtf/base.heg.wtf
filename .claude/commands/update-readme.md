# Update README Command

프로젝트의 README.md 파일을 생성하거나 업데이트합니다.

## 작업 순서

1. 프로젝트 루트에 README.md 파일이 존재하는지 확인
2. README.md가 존재하는 경우:
   - 기존 내용을 읽어 사용된 언어(한국어 또는 영어) 감지
   - 해당 언어를 유지하면서 업데이트
3. README.md가 존재하지 않는 경우:
   - 프로젝트의 주요 파일들을 분석하여 언어 감지 (주석, 문서 등)
   - 감지된 언어 또는 기본 한국어로 새로 생성
4. 프로젝트 구조 분석:
   - 주요 디렉토리 및 파일 구조 파악
   - package.json, pubspec.yaml, requirements.txt 등 의존성 파일 확인
   - 기술 스택 파악
5. README.md 내용 구성:
   - 프로젝트 제목 및 간략한 설명
   - 기술 스택
   - 설치 방법
   - 실행 방법
   - 프로젝트 구조 (필요시)
   - 라이선스 정보 (있는 경우)

## 언어 감지 규칙

- 기존 README.md가 있으면 해당 언어 유지
- 한글이 50% 이상이면 한국어로 판단
- 그 외에는 영어로 판단

## 주의사항

- 기존 README.md의 커스텀 내용은 최대한 보존
- 자동 생성된 섹션과 수동 작성 섹션을 구분하여 처리
- 민감한 정보(API 키, 비밀번호 등)는 포함하지 않음
- 과도한 정보보다는 핵심 정보 위주로 간결하게 작성
